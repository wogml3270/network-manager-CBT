## 프로젝트 본문

## 개요

네트워크관리사 2급 CBT는 시험 전 반복 풀이에 집중하기 위해 만든 개인 학습용 웹 애플리케이션입니다.

초기 아이디어는 학습 플래너, 오답노트, AI 해설, JSON 백업까지 포함한 비교적 큰 학습 도구였지만, 실제 사용 목적을 다시 정리하면서 범위를 과감하게 줄였습니다. 사용자가 가장 자주 하게 될 행동은 “문제를 고르고, 제한 시간 안에 풀고, 바로 결과와 해설을 확인하는 것”이었기 때문에 최종 방향은 CBT 전용 앱으로 잡았습니다.

기존에도 웹에서 사용할 수 있는 공식/기출 기반 CBT 서비스는 존재합니다. 다만 오래전에 PHP 기반으로 만들어진 형태라 화면 구성과 사용 경험이 현재 웹 애플리케이션의 UI/UX 흐름과는 거리가 있었습니다. 문제를 푸는 기능 자체는 가능하지만, 모바일 대응, 결과 확인, 공유, 반복 사용성 측면에서는 고리타분하게 느껴지는 부분이 있었습니다.

당장 네트워크관리사 2급 시험을 치르게 되었기 때문에 첫 번째 범위는 이 자격증 하나로 좁혔습니다. 다만 구조는 특정 시험에만 묶이지 않도록 CBT 설정, 문제은행, 채점, 결과 화면을 분리했고, 추후 다른 자격증 CBT로 확장할 수 있는 기반을 염두에 두었습니다.

## 문제 정의

이 프로젝트에서 가장 중요했던 제약은 공식 기출 원문을 그대로 제공하지 않는 것이었습니다.

네트워크관리사 2급 관련 자료는 공식 자료실에 기출 파일 형태로 제공되지만, 이를 서비스에 내장하거나 재배포하는 방식은 저작권 측면에서 적절하지 않다고 판단했습니다. 그래서 앱에는 공식 문장을 복제하지 않고, 시험 범위와 개념을 바탕으로 직접 작성한 유사문제 200문항을 넣었습니다.

또한 기존 CBT 서비스가 이미 존재한다는 점도 고려했습니다. 목표는 단순히 같은 문제풀이 화면을 다시 만드는 것이 아니라, 더 빠르게 진입하고 더 보기 좋게 풀고 더 자연스럽게 결과를 확인할 수 있는 경험을 만드는 것이었습니다.

또 하나의 제약은 비용이었습니다. OpenAI API 기반 해설 기능도 고려했지만, 지속적으로 유료 API를 호출해야 하는 구조는 개인용 학습 도구의 목적과 맞지 않았습니다. 결과적으로 모든 해설을 정적 데이터로 관리하고, 브라우저에서 바로 풀이가 가능하도록 만들었습니다.

## 주요 기능

## 확장 가능한 CBT 구조

현재는 네트워크관리사 2급만 지원하지만, 앱의 큰 구조는 다른 자격증 CBT를 추가할 수 있도록 나누었습니다.

문제은행은 JSON 파일로 분리했고, 과목 필터, 문항 수, 채점, 결과 화면은 데이터에 맞춰 동작합니다. 추후 다른 자격증으로 확장할 때는 새로운 문제 데이터와 시험 기준을 추가하는 방향으로 확장할 수 있습니다.

## CBT 설정

첫 화면은 마케팅 페이지나 대시보드가 아니라 바로 CBT 설정 화면입니다.

사용자는 과목과 문항 수만 선택하면 문제 풀이를 시작할 수 있습니다. 과목은 전체, 네트워크 일반, TCP/IP, NOS, 네트워크 운용기기로 나누었고, 문항 수는 20문항, 30문항, 50문항 중에서 선택할 수 있습니다.

기본 기준은 실제 시험 흐름에 맞춰 50문항, 50분, 60점 이상 합격으로 구성했습니다.

## 문제은행

문제 데이터는 `data/questions.json`에서 관리합니다.

각 문항은 문제, 선택지, 정답 인덱스, 상세 해설, 태그를 포함합니다. 추후 문제를 수정하거나 추가할 때 TypeScript 코드가 아니라 JSON 파일만 다루면 되도록 분리했습니다.

수록 범위는 다음 네 과목입니다.

- 네트워크 일반
- TCP/IP
- NOS
- 네트워크 운용기기

각 과목마다 50문항씩 총 200문항을 균등하게 구성했습니다.

## 랜덤 CBT 세션

매번 같은 문제 번호와 같은 선택지 순서로 나오면 반복 학습 효과가 떨어집니다.

이를 해결하기 위해 CBT를 시작할 때마다 문제 순서를 무작위로 섞고, 각 문제의 선택지 순서도 다시 섞습니다. 선택지가 섞인 뒤에도 정답 인덱스가 정확히 재계산되도록 별도 유틸 함수로 처리했습니다.

## 제출 전 확인

미응답 문항이 있는 상태에서 제출하면 `n번 문제를 선택하지 않았습니다. 제출하시겠습니까?` 형태의 확인 메시지를 띄웁니다.

사용자가 취소하면 첫 번째 미응답 문항으로 이동하고, 확인하면 그대로 채점합니다. 단순한 기능이지만 실제 CBT 사용 중 실수를 줄이는 데 꽤 중요했습니다.

## 결과와 해설

결과 화면은 점수, 정답 수, 응답 문항 수, 합격 여부를 먼저 보여줍니다.

이전에는 오답만 보여주는 구조였지만, 복습 관점에서는 맞힌 문제도 해설을 다시 읽을 수 있어야 했습니다. 그래서 최종 결과 화면에서는 전체 문항을 모두 렌더링하고, 각 문항마다 사용자가 선택한 답, 정답, 상세 해설을 함께 보여줍니다.

## 결과 공유

완료 후 지인에게 결과를 공유할 수 있도록 카카오톡 공유 기능을 추가했습니다.

로그인은 붙이지 않았습니다. 목적이 기록 저장이나 랭킹이 아니라 결과 공유였기 때문에, 별도 서버나 DB 없이 Kakao JavaScript SDK, Web Share API, 링크 복사 fallback으로 충분했습니다.

공유 링크는 `/share` 페이지로 연결되며, 점수와 정답 수, 합격 여부를 요약해서 보여줍니다.

## 배포 파이프라인

배포는 Vercel을 사용했습니다.

`main` 브랜치에 push하면 Vercel production 배포가 생성되고, GitHub Actions가 최신 READY 배포를 찾아 `network-manager-cbt.vercel.app` alias로 연결합니다. 기존 블로그 프로젝트에서 사용하던 alias 동기화 흐름을 이 프로젝트에도 맞게 이식했습니다.

필요한 환경변수는 `.env.example`에 정리했고, Kakao 공유에 필요한 JavaScript 키와 사이트 URL도 명시했습니다.

- 정답 인덱스 유효성
- 점수 계산과 합격 판정
- 문제/선택지 랜덤 배치

- 결과 화면 표시

- 공유 결과 페이지 렌더링

## 배운 점

이 프로젝트는 기능을 많이 넣는 것보다, 사용자가 실제로 반복할 핵심 행동만 남기는 것이 더 나은 선택일 때가 있다는 점을 다시 확인한 작업이었습니다.

초기 계획에는 AI 해설, 오답노트, 백업/복원, 학습 플래너가 있었지만 최종 앱에서는 모두 제외했습니다. 대신 문제 풀이, 채점, 해설, 공유라는 흐름은 훨씬 단단하게 만들 수 있었습니다.

또한 공식 자료를 다루는 서비스에서는 기능 구현만큼이나 저작권과 데이터 출처를 먼저 정리하는 것이 중요했습니다. 공식 원문을 그대로 내장하지 않고 자체 유사문제로 전환한 결정은 이 프로젝트의 방향을 명확하게 만들어준 핵심 선택이었습니다.

기존 CBT 서비스가 이미 존재하더라도, 사용 경험을 현대적으로 다시 설계하면 충분히 새 프로젝트의 의미가 생긴다는 점도 느꼈습니다. 단순히 기능을 복제하는 것이 아니라, 시험 직전 사용자가 더 편하게 반복할 수 있는 흐름으로 다시 만드는 것이 이 프로젝트의 핵심이었습니다.
